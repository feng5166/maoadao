## 01 · 产品总览 / PRD 总纲

- 一句话定位
- 产品要解决的真问题（孤独 / 情绪障碍人群的核心痛点）
- 不可替代性：为什么是硬件、为什么是"会记得你的猫"
- 价值主张（对用户的承诺）
- **产品价值观红线**（陪伴≠医疗、隐私是生死线、防依赖）
- Slogan 与品牌调性

## 02 · 用户与场景

- 目标人群分层：孤独人群 / 轻中度情绪困扰 / 临床情绪障碍（边界人群）
- 用户画像（2–3 个典型 persona）
- 核心使用场景：回家、深夜、低落时、日常陪伴、特殊日子
- 需求分层：在场感 → 被理解 → 被记得 → 被在乎

## 03 · IP 与人格设计（猫阿道）

- 基础人设：为什么是"不完美、需要被照顾"的猫
- 人格三层结构：出厂基底 / 用户可选性格 / 长期漂移
- 双向陪伴：让用户"也被需要"
- 人格的私有记忆（口头禅、专属称呼、专属习惯）
- 人格跟人走、不跟硬件走

## 04 · 核心功能设计

- Layer 1 · 在场感（主动招呼、微表情肢体、呼吸感待机）
- Layer 2 · 情绪识别与回应（多模态、分级回应、不评判）
- Layer 3 · 关系沉淀（记忆系统、情绪日历、专属仪式、成长可视化）
- Layer 4 · 主动关怀（异常察觉、不打扰式提醒）

## 05 · 个性化数据沉淀与记忆系统

- 数据四层：短期上下文 / 关系记忆 / 情绪长曲线 / 人格漂移参数
- 端云分层存储策略（游客端侧 vs 登录上云）
- 隐私原则：可删、可导出、敏感感知可物理关闭
- 换设备继承（人格不丢失）

## 06 · 情绪安全策略（红线）

- 危机信号识别（自伤 / 自杀倾向）
- 温柔转介专业资源的话术与触发逻辑
- 明确边界声明：陪伴非医疗替代
- 防依赖设计：鼓励走向真实关系

## 07 · 硬件形态设计

- 形态取舍：纯实体 vs 表情屏 vs 折中方案
- 触觉交互（可摸、可撸、有反馈）
- 声音设计：猫叫与拟人语音的比例
- 离线兜底：断网仍"活着"

## 08 · MVP 范围与优先级

- P0 / P1 / P2 功能清单
- 版本节奏（V1 验证陪伴内核 → V2 关系沉淀 → V3 主动关怀）
- 砍功能的原则

## 09 · 商业化与增长（后置）

- 硬件 + 订阅 / 内容 / 人格扩展
- 定价与价值锚点
- 关系留存如何转化为商业留存

## 10 · 指标体系（后置）

- 北极星指标候选
- 陪伴质量怎么度量（不是 DAU 那套）
- 安全相关的监控指标
