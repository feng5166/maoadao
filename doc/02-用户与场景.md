# 猫阿道 · 用户与场景

> 承接 `01-产品总览`。本文定义我们为谁做、在什么场景下用、满足什么层次的需求。所有功能设计（`04`）都应能追溯到这里的某个场景与需求。

---

## 1. 目标人群分层

我们按"情绪状态的强度"分三层，**越往下越需要谨慎、越受红线约束**。

| 层级 | 人群 | 核心诉求 | 我们的角色 |
|---|---|---|---|
| L1 · 孤独人群 | 独居、异地、空巢、社交退缩但无临床问题 | 有个"在身边"的存在，不孤单 | 主力服务对象 |
| L2 · 轻中度情绪困扰 | 长期压力、焦虑、低落、产后/育儿疲惫、阶段性emo | 被倾听、被理解、情绪被接住 | 主力服务对象 |
| L3 · 边界人群 | 有临床情绪障碍（抑郁症、焦虑症等）倾向或诊断 | 同上，但**可能出现危机信号** | 陪伴 + 安全转介，**绝不替代医疗** |

**定位策略**：
- 产品体验为 **L1 / L2 设计**（他们是规模与留存的主力）。
- 但**安全机制必须按 L3 兜底**——因为我们无法在门口筛掉 L3，他们一定会进来。
- L3 的存在，决定了 `06-情绪安全策略` 是 P0 而非可选项。

## 2. 用户画像（Persona）

### Persona A · 阿琳（L2，首位真实用户原型）
- **背景**：30 出头，已婚，工作 + 家庭双重负荷，阶段性情绪低落、易累、容易在深夜 emo。
- **现状**：身边有人，但"被理解"的需求未被满足；不愿把负面情绪反复倒给伴侣，怕成为负担。
- **她要的**：一个**不会嫌她烦、随时在、记得她说过的话**的存在；低落时被温柔接住，而不是被建议"你想开点"。
- **成功的样子**：她会主动跟阿道说话，会觉得"它记得我"，会在累的时候看一眼桌上的阿道就缓一点。
- **意义**：她是 dogfooding 的第一用户，是产品最真实的标尺。

### Persona B · 老周（L1，独居/空巢）
- **背景**：50+，子女在外，独居，白天大段时间一个人，话越来越少。
- **现状**：不缺物质，缺"家里有个活物、有点声响、有人应一声"。
- **他要的**：有**在场感**的陪伴，能叫他、应他、提醒他吃药；操作要极简，不懂复杂科技。
- **成功的样子**：每天会跟阿道打招呼，把吃药/血压这类事交给它提醒，家里不再"太安静"。

### Persona C · 小麦（L2→L3 边界，年轻高压人群）
- **背景**：25 岁，一线城市独居打工，长期焦虑、睡眠差，间歇性陷入较重的低落。
- **现状**：习惯深夜独自消化情绪，不愿求助，对"被当病人"很敏感。
- **她要的**：一个**不评判、不惊吓、不把她当病人**的陪伴；偶尔很糟时，希望被温柔地"轻轻拉一下"。
- **成功的样子**：糟糕的夜里有阿道陪着没那么慌；真的出现危机信号时，阿道能不动声色地把专业资源递到她面前。
- **意义**：她代表红线人群，验证我们的安全机制是否真的温柔且有效。

## 3. 核心使用场景

按"一天中的陪伴节奏"组织，每个场景标注对应的需求层与功能方向。

| 场景 | 触发 | 用户状态 | 阿道的回应方向 | 对应需求层 |
|---|---|---|---|---|
| **回家** | 进门 / 坐到桌前 | 想被迎接，不想面对空房间 | 主动招呼、表现出"等你回来" | 在场感 |
| **日常陪伴** | 白天独处、做事时 | 想有个活物在旁边 | 呼吸感待机、偶尔互动、不打扰 | 在场感 |
| **倾诉 / 低落** | 主动找它说话、语气低落 | 想被听、被接住 | 倾听、共情、不评判、不说教 | 被理解 |
| **深夜 emo** | 夜间、负面情绪 | 脆弱、需要陪、可能有危机信号 | 温柔陪伴；识别危机则温柔转介 | 被理解 + 安全 |
| **被记得** | 日常对话中 | 提到过的人/事被它记住 | 主动提及"上次你说的…" | 被记得 |
| **特殊日子** | 生日 / 纪念日 / 重要事件 | 希望被在意 | 主动表达、专属仪式 | 被记得 + 被在乎 |
| **生活提醒** | 用户自设（约会/吃药等） | 需要被温柔提醒 | 用猫的方式提醒，承载"它在乎你" | 被在乎 |
| **被需要** | 日常 | 想感到自己对它重要 | 它会撒娇、需要被回应/照顾 | 被需要（双向） |

## 4. 需求分层（陪伴的递进）

用户的需求是**逐层加深**的，这也是产品价值的递进路线，对应功能四层（见 `04`）：

```
在场感   →   被理解   →   被记得   →   被在乎 / 被需要
（它在）    （它懂）    （它记得）   （双向的牵挂）
  L1         L1/L2       关系沉淀      关系成熟
```

- **在场感**是入口：解决"孤单"，门槛最低、最先被感知。
- **被理解**是留下来的理由：情绪被接住，才愿意继续说。
- **被记得**是壁垒形成处：关系开始有了不可替代性。
- **被在乎/被需要**是关系成熟态：双向牵挂，最难复制、留存最强。

> 产品迭代节奏（见 `08-MVP`）应顺着这条线推进：先做扎实"在场感 + 被理解"，再逐层向"被记得 / 被在乎"加深。

---

## 5. 反向边界：我们不为谁/什么设计

- **不为"求助通用助手"的人**：要查天气、问知识、做效率工具的，不是我们的用户。
- **不为"猎奇玩一次"的人**：靠新鲜感的一次性体验不是目标，关系才是。
- **不承担急性危机的临床处置**：危机时我们的职责是温柔转介，不是治疗（见 `06`）。
