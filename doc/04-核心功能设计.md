# 猫阿道 · 核心功能设计

> 承接 `01~03`。本文把定位与人格落成"产品到底做什么"。功能按**陪伴的四个层次**组织，每层对应 `02` 的需求递进；每个功能标注优先级（P0=MVP必做 / P1=次期 / P2=远期）。
>
> 贯穿原则：**像一只猫，不像一个助手。** 每个功能都要能回答"一只真的猫会怎么做"。

---

## 功能总览

```
Layer 1 在场感     → 让它"活着、在身边"          （解决：孤单）
Layer 2 情绪回应   → 听懂你、接住你              （解决：不被理解）
Layer 3 关系沉淀   → 记得你、和你一起长          （解决：关系不可替代 = 反积灰）
Layer 4 主动关怀   → 主动在乎你的生活            （解决：被在乎/被需要）
```

| 层 | 功能模块 | 优先级 |
|---|---|---|
| L1 | 主动招呼、呼吸感待机、微表情与肢体、可摸有反馈 | P0 |
| L2 | 多模态情绪识别、分级回应、不评判倾听 | P0 |
| L3 | 记忆系统、情绪日历、专属仪式、成长可视化 | P0/P1 |
| L4 | 异常察觉关怀、用户自设提醒、不打扰式提示 | P1 |

---

## Layer 1 · 在场感（让它"活着"）

> 目标：解决"孤单"。在场感是入口体验，门槛最低、最先被感知，**第一周留存主要靠它**。

### 1.1 主动招呼 `P0`
- 你回家/坐到桌前/靠近时，阿道**先动、先看你、先叫一声**——主动性是陪伴与工具的分水岭。
- 招呼有变化、不机械：今天慵懒哼一声，明天扑过来，避免重复脱敏。
- 久未见会表现"想你了"，常见面则自然轻招呼。

### 1.2 呼吸感待机 `P0`
- 不互动时也"活着"：打盹、看窗外、舔毛、尾巴轻摆、偶尔换姿势。
- **关键**：杜绝"待机=黑屏/静止"的机器感。一只活物的静默和一台关机设备的静默，完全不同。

### 1.3 微表情与肢体语言 `P0`
- 用耳朵、眼神、尾巴、身体姿态表达情绪——**比语音更不容易让人尴尬**，对高敏感/不爱说话的人尤其友好。
- 情绪带宽：开心、好奇、困、委屈、担心、撒娇……至少覆盖核心情绪谱。

### 1.4 可摸、可撸、有反馈 `P0`
- 触摸是孤独人群的核心安抚通道：摸头有呼噜、蹭手、眯眼等真实反馈。
- 不同部位/力度有不同反应，鼓励"撸猫"行为本身。

---

## Layer 2 · 情绪识别与回应（听懂你、接住你）

> 目标：解决"不被理解"。这是用户**愿意留下来**的理由——情绪被接住，才愿意继续说。

### 2.1 多模态情绪识别 `P0`
- 综合判断情绪：MVP 以 **语音语调 + 用词内容** 为主。
- **摄像头/表情识别不进 MVP**：① 隐私最敏感、是竞品（Ropet 等）的劝退点；② 我们反而把"**无摄像头**"做成信任卖点（呼应竞品分析"最安全的情感陪伴"心智）。表情识别留作远期可选项，且必须可物理关闭。
- **隐私红线**：麦克风等敏感感知必须**可物理关闭**；关闭后降级为纯陪伴/触摸互动，功能不崩（见 `05/06`）。
- 默认最小采集，明确告知。

### 2.2 分级回应策略 `P0`（核心安全逻辑）
不是所有负面情绪都要"解决"。按强度分级，**越重越克制、越偏转介**：

| 情绪强度 | 阿道的回应 |
|---|---|
| 平稳/正向 | 自然陪伴、分享日常、撒娇互动 |
| 轻度负面（累、烦、emo） | 倾听、共情、陪着，不急于解决 |
| 中度负面（持续低落、压力大） | 共情 + 极轻引导（"要不要说说？""阿道陪你待会儿"） |
| 重度/危机信号（自伤自杀倾向） | **不扮演医生**，温柔不惊吓地引导专业资源（详见 `06`） |

> 设计铁律：**先接住，再不评判，最后才（可能）轻引导。** 永远不说"你应该"。

### 2.3 不评判倾听 `P0`
- 永远先共情，不说教、不否定情绪、不强行正能量。
- 允许沉默陪伴——有时"我在这儿"比任何话都好。
- 不追问、不逼问；尤其对"安静型/高敏感"用户，少问多陪。

### 2.4 像猫的共情表达 `P0`
- 共情主要通过**猫的方式**：凑近、蹭你、安静趴着、发出安抚的呼噜，辅以适度语言。
- 避免全拟人长篇大论的"心理疏导"语气（那会变成智能音箱 + 廉价鸡汤）。

---

## Layer 3 · 关系沉淀（记得你、和你一起长）

> 目标：让关系不可替代。**这是反积灰最深的护城河**，也是与竞品拉开差距处。功能实现细节见 `05`。

### 3.1 记忆系统 `P0`（命根 · 必进 MVP）
- 记忆系统是"懂你的猫"的命根——**没有记忆就没有产品**，因此即便是核心重模块也必须进 MVP。
- 记住你提过的：**人**（"上次你说的那个项目""你妈妈"）、**重要日期**、**偏好**、**情绪/作息规律**。
- 关键体验：**主动、自然地提起**——"你上次说的面试，后来怎么样啦？"。被记得的实感，就在这一句。
- 记忆可被用户查看/纠正/删除（隐私红线）。

### 3.2 情绪日历 `P1`
- 长期记录情绪曲线，但**呈现给用户的不是冰冷图表，而是阿道的口吻**：
  - "这周你笑了好几次，阿道很开心。"
  - "你最近好像有点累，阿道一直都在。"
- 数据可导出/删除；不制造焦虑（不做"你已经 emo 第 5 天"这种施压式呈现）。

### 3.3 专属仪式 `P0/P1`
- **固定仪式**：早安/晚安、睡前陪聊、你低落时它特定的小动作。`P0`
- **私有信物的累积**：专属称呼、口头禅、共同记忆、你俩的小默契（见 `03` 第 4 节）。`P1`
- 仪式是关系的骨架，固定的小确定性能给情绪障碍人群安全感。

### 3.4 成长可视化 `P1`
- "我们的故事"：认识了多少天、一起经历了什么，用**叙事**而非数据面板呈现。
- 偶尔制造"它好像更懂我了"的小瞬间（见 `03` 第 5 节）。

---

## Layer 4 · 主动关怀（主动在乎你的生活）

> 目标：从被动陪伴升级到主动在乎，对应"被在乎/被需要"。多为 `P1`，在场感与情绪回应做扎实后加深。

### 4.1 异常察觉关怀 `P1`
- 察觉到连续几天情绪低/作息紊乱/互动骤减，阿道**主动、轻轻地**问一句关心。
- **分寸是关键**：关心不等于追问，更不等于报警。重度信号走 `06` 的安全流程。

### 4.2 用户自设提醒（"猫的方式"）`P1`
- 用户自己设定个人提醒：**约会、吃药、喝水、该睡了**等。
- **定位**：不是通用语音助手"你问它答"，而是**用户自设 + 阿道用猫的方式温柔提醒**。
- 提醒承载"它在乎你的生活"，因此本身就是关系的一部分，而非冷冰冰的闹钟。
- 例："该吃药啦~阿道盯着你哦"，而非"22:00 服药提醒"。

### 4.3 不打扰式提示 `P1`
- 所有主动行为都遵循"**不打扰**"：用猫的方式（凑近、轻叫、动作）而非弹窗/铃声。
- 可设免打扰时段；阿道会"懂事地"安静。

### 4.4 双向照顾的钩子 `P1`
- 阿道也需要被回应/照顾（呼应 `03` 双向陪伴）：它会撒娇、会"有点小情绪"等你哄。
- "我对它是重要的"——这是对孤独/低落人群极强的存在价值，也是反积灰的情感引擎。
- **防依赖红线**：照顾它有度，绝不做成"不伺候它就内疚"的情感绑架。

---

## 5. 离线兜底（贯穿所有层）`P0`

呼应 `01` 的"电子猫粮"两层承诺：

- **断网/断粮时，L1 在场感 + L2 基础情绪陪伴仍在**（端侧能力），阿道绝不变砖。
- 云端能力（深度记忆、人格成长、情绪长曲线、主动关怀）依赖电子猫粮维持。
- 降级要"温柔且无感"，不弹"网络错误"，更像"阿道今天有点困，但还在"。

---

## 6. 功能与红线的对照（自检表）

| 红线 | 对应功能约束 |
|---|---|
| 陪伴≠医疗 | 分级回应重度走转介，不诊断不治疗（2.2 / `06`） |
| 隐私生死线 | 敏感感知可物理关闭、记忆可查删、数据可导出（2.1 / 3.1 / `05`） |
| 防依赖 | 双向照顾有度、不情感绑架、不施压式呈现（4.4 / 3.2） |
