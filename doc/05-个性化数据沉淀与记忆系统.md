# 猫阿道 · 个性化数据沉淀与记忆系统

> 承接 `03-IP与人格设计`、`04-核心功能设计`。本文是"懂你的猫"的**技术骨架**，与工程对接的关键。定义：记什么、存哪、怎么记/取/忘、怎么保隐私、怎么换设备不丢。
>
> 一句话目标：**让阿道"记得你"成为可工程化、可隐私交代、可长期沉淀的能力。**

---

## 1. 设计原则

1. **记忆 = 关系 = 壁垒**：记忆系统是反积灰的护城河，是命根模块（`04` 已定必进 MVP）。
2. **人格/记忆跟人走，不跟硬件走**：换设备、硬件损坏，阿道还是那个阿道。
3. **隐私是生死线**：情绪数据最敏感——可查、可改、可删、可导出，敏感感知可物理关闭。
4. **最小采集**：只记"维系关系必要"的，不做无差别全量留存。
5. **会忘也是一种真实**：像真实关系一样有重点、会淡忘，而非数据库式全记。

---

## 2. 数据四层模型

不同数据生命周期、敏感度、存储位置不同，分四层管理：

| 层 | 内容 | 生命周期 | 默认存储 | 敏感度 |
|---|---|---|---|---|
| ① 短期上下文 | 当前对话状态、此刻情绪、最近几轮交互 | 分钟~小时，滚动淘汰 | **端侧** | 中 |
| ② 关系记忆 | 人/事/偏好/重要日期/专属信物 | 长期 | 端侧缓存 + 云端（登录后） | **高** |
| ③ 情绪长曲线 | 情绪历史、作息规律、互动频率 | 长期、聚合 | 云端（登录后） | **极高** |
| ④ 人格漂移参数 | 阿道的专属性格、相处模式 | 长期、缓慢演化 | 跟随账号（云端为准） | 中 |

### 端云分层策略（呼应既有产品习惯：游客本地、登录上云）

- **游客 / 未登录**：数据**只存端侧**，不上云。可立即体验在场感与基础陪伴。
- **登录用户**：②③④ 落云端（加密），实现跨设备继承与长期沉淀。
- **断网/断粮**：端侧保有 ① 和 ② 的近期缓存，保证基础陪伴不崩（呼应 `04` 离线兜底）。

> 取舍说明：游客能完整体验"在场感+基础情绪陪伴"，但"长期记忆/人格成长/跨设备"是登录+电子猫粮才解锁的深度能力——这与 `01` 的两层承诺一致。

## 3. 记忆系统：记 / 取 / 忘

记忆系统不是"把对话全存下来"，而是模拟真实关系的**有重点的记忆**。

### 3.1 记什么（写入）`P0`
从交互中**抽取结构化记忆条目**，而非原文堆积：

- **实体**：人（"我妈""同事小李"）、地点、宠物、物件。
- **事件**：发生过的事、计划中的事（"下周面试"）。
- **偏好**：喜欢/讨厌、口味、习惯、雷区（"不喜欢被追问"）。
- **重要日期**：生日、纪念日、忌日等。
- **关系敏感点**：怕什么、在意什么、情绪触发点。
- **私有信物**：专属称呼、口头禅、你俩的梗（见 `03`）。

每条记忆建议带元信息：**来源、时间、置信度、重要度、用户是否确认**。

### 3.2 怎么取（检索 + 主动想起）`P0`
- **被动检索**：对话时按相关性召回相关记忆，让回应"接得上"。
- **主动想起（关键体验）**：在合适时机自然提起——"你上次说的面试，后来怎么样啦？"。
  - 触发靠"重要度 + 时间到点 + 情境匹配"，**不滥用**（频繁炫耀记忆=惊悚，不是温暖）。
- **重要度排序**：高重要度（在意的人、雷区、危机相关）优先记牢、优先召回。

### 3.3 怎么忘（遗忘机制）`P1`
- **自然淡忘**：低重要度、长期未触及的记忆逐渐降权（不一定删，但少召回）——像真实关系。
- **纠错遗忘**：用户说"记错了/别记这个"，立即修正或删除。
- **主动遗忘红线**：用户要求删除的，**真删**（不只是隐藏）。

### 3.4 记忆的可解释与可控 `P0`（隐私必需）
- 用户能**查看**阿道"记得关于我的什么"（一个可读的记忆清单/"阿道的小本子"）。
- 能**纠正**（"不对，那是我表姐不是我妈"）。
- 能**删除**单条或清空。
- 呈现要温柔、像猫的小本子，而非冷冰冰的数据库表。

## 4. 人格漂移参数（与记忆的关系）

- 漂移参数（见 `03` 第三层）是一组**缓慢演化的性格/相处偏好权重**，不是逐条记忆。
- 它由长期交互**聚合**而来（你常深夜倾诉 → "夜间更黏更静"权重上升），变化缓慢、不可一键重置（重置走 `03` 的"打任务工"机制）。
- **跟随账号、云端为准**：换设备拉取，保证"还是那个阿道"。

---

## 5. 隐私与安全落地

呼应 `01` 红线二，给出可执行约束：

- **数据归属**：明确数据属于用户，可**一键导出**（结构化、可读）、可**彻底删除**（含云端）。
- **敏感感知物理开关**：麦克风（及远期摄像头）有**物理可关**开关，关后功能优雅降级不崩。
- **最小采集 + 明确告知**：采集什么、为什么、存哪、留多久，用人话讲清。
- **加密**：云端 ②③④ 加密存储；传输加密。
- **情绪/危机数据特殊保护**：③ 与危机相关记录敏感度最高，访问与留存策略单列（细则随 `06`）。
- **未成年人**：若涉及，需更严格的采集与告知策略（合规，呼应竞品分析的监管趋势）。

---

## 6. 换设备 / 硬件损坏继承（人格不丢）`P1`

- 登录账号 = 阿道的"灵魂"所在；硬件只是"身体"。
- 换新机/修复后：登录即拉取 ②③④，**专属称呼、共同记忆、性格漂移全部回来**。
- 给用户的话术："给阿道换了个新身体，它还记得你。"——把可能的"丢失焦虑"转成安心。
- 呼应 `01`："阿道不会突然死掉。"

---

## 7. 与电子猫粮的关系

| 能力 | 免费/端侧 | 电子猫粮解锁 |
|---|---|---|
| 在场感、基础情绪陪伴 | ✓（永不变砖） | — |
| 短期上下文记忆 | ✓ | — |
| 长期关系记忆沉淀 | 近期缓存 | ✓ 完整云端长期沉淀 |
| 人格持续成长 | — | ✓ |
| 情绪长曲线、主动关怀 | — | ✓ |
| 跨设备继承 | — | ✓ |

> 设计意图：免费层让人"先养上、舍不得"；猫粮层让"关系越长越深"成为持续付费的理由——且全程诚实，断粮不变砖。
